# Contributing to Korda

Thank you for helping improve Korda.

1. Open an issue describing the problem or proposed behavior.
2. Create a focused branch and preserve the local-first security boundaries.
3. Run `npm ci`, `npm test` and `npm run build`.
4. For terminal, browser, workspace or broker changes, add a focused regression test.
5. Open a pull request explaining user impact, verification and any remaining risk.

Do not commit credentials, terminal transcripts, private workspace files, generated packages or local runtime directories. Contributions intentionally submitted to this repository are accepted under Apache-2.0 unless explicitly stated otherwise.
